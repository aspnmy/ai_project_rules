# AI项目规则管理系统

这是一个专为AI辅助代码开发项目设计的规则管理系统，提供了一套完整的项目规范、自动化工具和同步机制。

## 🎯 项目概述

本项目旨在为AI辅助代码开发提供标准化的规则管理体系，确保代码质量、项目一致性和团队协作效率。通过自动化工具和定期同步机制，让开发者始终使用最新的项目规范。

## 📋 主要功能

### 1. 规则文件管理
- **project_rules.md**: 核心规则文件，包含完整的项目规范
- **自动化拉取**: 定期从GitHub同步最新规则
- **版本控制**: 支持规则文件的版本管理和回溯
- **备份机制**: 自动备份历史版本，支持快速恢复

### 2. 规则分类体系
- **高优先级规则** (🔴): 必须严格遵守的强制性规范
- **中优先级规则** (🟡): 建议遵循的最佳实践
- **低优先级规则** (🟢): 参考性建议和优化建议

### 3. 冲突解决机制
- **优先级层次表**: 明确的规则优先级排序
- **自动冲突检测**: 智能识别规则冲突
- **解决方案建议**: 提供标准化的冲突处理流程

## 📖 规则体系详解

### 项目代码修改规则
- 所有代码修改必须先提交到git仓库
- 平台适配需要新建dev-platform分支
- 严禁直接修改master/main分支

### 临时文件管理规则
- 临时文件必须使用`temp-${filename}`前缀
- 严禁将临时文件提交到版本控制
- 调试完成后必须清理临时文件

### 文件路径使用规则
- 所有文件调用必须使用相对路径
- 禁止从绝对路径加载资源
- 必须获取当前文件路径作为基点

### 文本版本控制规则
- 使用`**-${filename}-d${num}`格式进行版本管理
- 关联性功能扩展使用`temp-${filename}`
- 版本文件在提交前需要合并和清理

### 语法规范自动调整规则
- IDE规范提示高于现有语法规范
- 修改语法规范时需要标明对应版本号
- 确保语法规范的版本兼容性

### 规则优先级漏洞逻辑自动调整规则
- 每次修改规则文件后必须检查优先级漏洞
- 出现冲突或逻辑漏洞时需要重新调整
- 确保规则体系的完整性和一致性

## 🔧 自动化工具

### 规则拉取器 (rule_puller.py)
- **自动同步**: 每72小时自动检查更新
- **智能备份**: 更新前自动备份当前版本
- **冲突检测**: 比较文件内容，避免重复更新
- **错误恢复**: 更新失败时自动回滚

### 配置管理
- **JSON配置**: 使用配置文件管理同步设置
- **灵活调整**: 支持自定义同步间隔和规则
- **状态监控**: 实时监控规则文件状态

## 🛠️ 开发指南

### 规则文件结构
```
project_rules.md
├── 规则优先级说明
├── 项目代码修改规则
├── autotest规则
├── 临时文件管理规则
├── 文件路径使用规则
├── 文件路径下代码编辑规则
├── 文本版本控制规则
├── Git提交远程仓库前公用规则
├── 语法规范自动调整公用规则
├── 规则优先级漏洞逻辑自动调整规则
└── 代码语法规范
```

### 优先级标识
- 🔴 **高优先级**: 必须遵守的强制性规则
- 🟡 **中优先级**: 建议遵循的规范性要求
- 🟢 **低优先级**: 参考性的最佳实践

### 冲突处理原则
1. **IDE语法错误** > 所有其他规则
2. **IDE编译错误** > 代码规范
3. **公用规则** > 代码规范
4. **Git规则** > 文本版本控制
5. **具体规则** > 通用规则
