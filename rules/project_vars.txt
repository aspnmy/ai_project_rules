# 此处登记本工作区中使用的私有全局变量
# 格式结构为：${var_name}|${var_type}|${var_title}
# ${var_name} 代表项目统一的变量名，在本项目中全局统一化使用规范常用英文，能够让AI主动识别到正确的含义
# ${var_type} system_path_var 、 debug_var 、project_var
# system_path_var代表此变量类型使用系统环境变量读取，${var_title}直接记录为nil，读取方式可以复用或参考db_lists.txt中的系统变量读取方法
# debug_var 代表此变量类型为调试所用默认值，可以明文登记在${var_title}中，初始化时在工作区新增debug_vars.json 内容如下 {"debug_vars":["${var_name01}":"${var_title01}","${var_name02}":"${var_title02}"]}
# json文件默认让Ai更加描述配置文件自行生成，尽量不做人为干预
# project_var 主要登记本工作区中主要的业务变量，比如规则文件名路径
# 变量前缀规范：常用约定 PG_ 代表本工作区全局变量 ； priv_${filename}_${function_designation} 代表私有变量${filename}代表此变量用于哪个文件名中、${function_designation}代表变量的功能名称，便于快速从注释中定位，真实的变量名应该是 priv_${filename}_${function_designation}
