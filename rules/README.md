# rules目录说明

rules目录是项目的规则文件存放目录，包含项目的核心规则定义、文件规则和项目变量配置。

## 📁 目录文件说明

### 核心规则文件
- **project_rules_backup.txt**: 项目规则文件的备份版本
- **project_vars.txt**: 项目变量配置文件，定义全局变量
- **files_rules.txt**: 文件路径规则记录，定义文件命名和分类规则
- **update_rules.txt**: 更新规则配置文件

### 数据库相关配置
- **db_lists.txt**: 数据库列表配置文件

## 📝 规则文件说明

### project_rules_backup.txt
- 项目核心规则文件的备份版本
- 用于在主规则文件损坏或需要回滚时恢复
- 必须与主规则文件保持同步更新

### project_vars.txt
- 定义项目中使用的全局变量
- 变量类型包括system_path_var、debug_var和project_var
- 使用`${var_name}|${var_type}|${var_title}`格式定义
- 新增变量必须在此文件中注册

### files_rules.txt
- 记录项目中各种文件的命名规则和分类
- 包括生产代码、测试代码、临时文件等分类
- 定义文件的功能描述和使用场景
- 记录docs文件与csdn_md文件的对应关系

### update_rules.txt
- 定义规则更新的策略和方法
- 配置规则拉取和同步的参数

### db_lists.txt
- 定义数据库列表和相关配置
- 支持系统环境变量的读取和使用

## 🔒 规则文件管理

- 规则文件的更新必须遵循项目规范
- 重要规则文件必须定期备份
- 规则变更需要记录变更历史
- 请不要直接修改规则文件，应通过规则管理器进行更新
